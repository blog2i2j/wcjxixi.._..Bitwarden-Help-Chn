## 推进 Bitwarden 发展 <a href="#advancing-bitwarden" id="advancing-bitwarden"></a>

Bitwarden 不断新增功能，并集成了丰富的特性，以保障企业安全并让管理员倍感满意。

### 保存并分享任何内容 <a href="#save-and-share-anything" id="save-and-share-anything"></a>

企业如何使用存储的项目：

* 登录和通行密钥 - 网站、App、服务器、路由器、SSH 密钥
* 支付卡 - 公司信用卡、购物卡（P 卡）
* 身份 - 收货地址、邮件收发点、高管差旅信息（供助理使用）
* 安全笔记 - 敏感信息、安全流程、备份代码

### 无需密码即可快速访问您的密码库 <a href="#access-your-vault-quickly-without-passwords" id="access-your-vault-quickly-without-passwords"></a>

* 设备登录
* 受信任设备 SSO
* 通行密钥登录
* 桌面端、移动端 App 及浏览器扩展的生物识别解锁
* PIN 码解锁

### 让每个人都能轻松访问网站 <a href="#make-accessing-websites-easy-for-everyone" id="make-accessing-websites-easy-for-everyone"></a>

* 激活自动填充策略 - 在网页加载时自动填充登录
* 使用 SSO 策略自动登录 - 在身份提供程序的仪表板中使用快捷方式，快速访问网站并实现统一的 SSO 体验

### 允许用户自行批准新的 SSO 登录 <a href="#let-users-self-approve-new-sso-logins" id="let-users-self-approve-new-sso-logins"></a>

* 受信任设备 SSO 允许用户无需密码即可登录
* 用户可通过其他已登录的已信任 Bitwarden App 自行批准新的登录请求
* 支持从网页端、移动端、桌面端 App 及浏览器扩展进行批准

### 使用策略自定义您的组织 <a href="#customize-your-organization-with-policies" id="customize-your-organization-with-policies"></a>

* 账户恢复管理策略
* 会话超时策略
* 设置其他安全要求，如主密码和密码生成器复杂度

### 决定谁可以创建和访问集合 <a href="#decide-who-can-create-and-access-collections" id="decide-who-can-create-and-access-collections"></a>

使用集合管理设置来决定集合的工作方式。注意：仅可由组织所有者调整。

* 选择允许管理员和所有者访问组织中的所有项目
* 决定创建或删除集合是否只能由管理员操作
* 赋予用户直接自助服务的权限

### 增强您组织的安全性 <a href="#enhance-security-for-your-organization" id="enhance-security-for-your-organization"></a>

设置这些策略以实现更严格的控制

* 账户恢复管理
* 单一组织
* 禁用导出
* 要求单点登录身份验证
* 集中化组织所有权

然后

* 声明您的域名
* 与 SIEM 工具集成
* 运行 Access Intelligence 报告

### 为内部页面分配 URI 及 URI 匹配规则 <a href="#assign-uris-and-uri-matching-for-internal-pages" id="assign-uris-and-uri-matching-for-internal-pages"></a>

URI（URL 或移动 App 标识符）用于验证自动填充的页面

* 创建自定义 URI 控件，以调整自动填充匹配的工作方式
* 在指定域名上屏蔽自动填充运行（兼容性）

### 为员工提供 Bitwarden <a href="#give-bitwarden-to-your-employees" id="give-bitwarden-to-your-employees"></a>

* 每个组织成员均可免费获得 Bitwarden 家庭版方案
* 非成员（如工厂工人）也可获得赞助的方案
* 在工作之外强化良好的安全实践

### 集成您的技术栈 <a href="#integrate-your-tech-stack" id="integrate-your-tech-stack"></a>

配置 Bitwarden 以使用您现有的技术基础设施

* 支持目录集成、SCIM、SSO 身份提供程序、托管设备、SIEM
* 使用公共 API 和密码库管理 API 创建您自己的集成

## 2025 年新功能 <a href="#new-in-2025" id="new-in-2025"></a>

选择 Bitwarden 在 2025 年的创新功能和新业务特性。

### Access Intelligence <a href="#access-intelligence" id="access-intelligence"></a>

包含在所有企业版订阅中，针对与高优先级应用相关的风险凭证采取行动。发现成员正在使用的影子 IT 和未授权应用，按应用优先级解决风险，引导员工更改密码，并跟踪安全改进情况。

了解更多：Access Intelligence

### 密码库健康警报与密码指导 <a href="#vault-health-alerts-and-password-coaching" id="vault-health-alerts-and-password-coaching"></a>

用户可直接在其密码库中收到关于凭据安全问题的主动通知。Bitwarden 会将用户重定向至网站的修改密码表单，并协助生成及保存新的安全密码。

了解更多：更改存在风险的密码

### 集中化数据所有权策略 <a href="#centralize-data-ownership-policy" id="centralize-data-ownership-policy"></a>

确保 Bitwarden 中保存的所有项目均归组织所有。用户会在组织密码库中看到一个**我的项目**位置。

* 支持完整的报告
* 当员工退出组织时，管理员将获得对用户**我的项目**的访问权&#x9650;**。**

了解更多：企业策略

### 将 Bitwarden 与 AI 结合使用 - Bitwarden MCP 服务器 <a href="#use-bitwarden-with-ai-bitwarden-mcp-server" id="use-bitwarden-with-ai-bitwarden-mcp-server"></a>

模型上下文协议 (MCP) 服务器是密码管理实现安全 AI 认证的基础。通过它可编写纯文本命令与 Bitwarden 交互，专为本地自托管 Bitwarden 服务器设计。

了解更多：Bitwarden 借助 MCP 服务器奠定安全 AI 认证基础

### 集合管理设置的改进 <a href="#improvements-to-collection-management-settings" id="improvements-to-collection-management-settings"></a>

新增事件日志记录与更清晰的命名方式，使设置对组织的影响更易理解。此外，新增选项允许所有者决定是否允许拥有「管理集合」权限的成员从组织密码库中彻底删除条目。

了解更多：集合设置

### 新增企业策略 <a href="#new-enterprise-policies" id="new-enterprise-policies"></a>

已新增更多企业策略，以进一步控制 Bitwarden 组织的运作方式。

* 阻止在已声明域名下创建账户
* 移除卡片项目类型
* 移除使用 PIN 码解锁
* 默认 URI 匹配检测
* 移除免费的 Bitwarden 家庭赞助

了解更多：企业策略

### 使用通行密钥登录 <a href="#log-in-with-passkeys" id="log-in-with-passkeys"></a>

通过支持 WebAuthn PRF 扩展的任何通行密钥（如 YubiKey）快速访问 Bitwarden 密码库

* 支持在网页应用和 Bitwarden 浏览器扩展中使用
* 无需用户名、密码或两步登录验证——是管理员应急账户的绝佳选择。

了解更多：使用通行密钥登录

### 浏览器直接导入 <a href="#browser-direct-import" id="browser-direct-import"></a>

直接从基于 Chromium 的浏览器导入凭证至 Bitwarden 密码库，无需导出和管理 CSV 文件。提升用户上手体验与管理员部署效率。需使用 Bitwarden 桌面应用程序。

了解更多：直接从浏览器导入

### 简化的 SSO 登录 <a href="#streamlined-sso-login" id="streamlined-sso-login"></a>

简化的单点登录认证流程，可减少登录步骤并提升用户体验。必须使用 SSO 登录的用户，其他登录选项将显示为灰色不可用状态。

### SSH 代理 <a href="#ssh-agent" id="ssh-agent"></a>

Bitwarden SSH 代理内置于 Bitwarden 桌面应用程序中，可用于服务器身份验证、Git 提交签名以及与其他基于 SSH 的服务进行交互。这既简化了开发人员的工作流程，又保持了集中化组织所有权。

了解更多：SSH 代理

### 批准来自其他设备的 SSO 登录 <a href="#approve-sso-logins-from-other-devices" id="approve-sso-logins-from-other-devices"></a>

使用受信任设备的 SSO 时，用户可通过已认证的 Bitwarden 网页端、移动端和桌面端应用及浏览器扩展，自行批准新设备登录。这无需 IT 干预即可实现灵活安全的设备验证。

了解更多：批准受信任设备

### 声明账户 <a href="#claimed-accounts" id="claimed-accounts"></a>

当企业组织声明域名后，已加入组织的成员若使用匹配的域名（例如 jdoe@mycompany.com）的电子邮箱地址，其账户将被组织声明，从而实现对成员账户的更强管控。

了解更多：声明域名
